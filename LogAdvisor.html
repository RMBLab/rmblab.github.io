Learning to Log Exceptions

Jieming Zhu1,  Pinjia He1,  Qiang Fu2,  Michael R. Lyu1,  and Dongmei Zhang2
1The Chinese University of Hong Kong      2Microsoft Research Asia

Introduction

Logging is a common programming practice of vital importance, as it directly determines the quality of collected logs for postmortem analysis. Optimal logging that balances utility and overhead is desired, by covering necessary runtime information yet not incurring large overhead or producing too many trivial logs. However, current research has little focused on studying the optimal logging practices, e.g., how to help developers make informed logging decisions. Even in a leading software industry like Microsoft, there is a lack of rigorous specifications for developers to guide their logging activities. As a result, logging has become an important yet tough decision during development that mostly depends on the domain knowledge of developers. To fill this gap, in this paper, we propose "learning to log", which aims to automatically learn the common logging practices from existing code repositories. As a proof of concept, we provide the design and implementation of LogAdvisor, an automatic suggestion tool for exception logging, which can immensely help developers make informed logging decisions and reduce their effort on logging. We evaluate LogAdvisor on both commercial software systems from Microsoft and open-source software systems from GitHub (totally 19.1M LOC and 100.6K logging statements). The promising results demonstrate the feasibility and efficacy of our logging suggestion tool, which can serve as a significant step towards automatic logging.
