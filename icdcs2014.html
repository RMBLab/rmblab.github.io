Towards Online, Accurate, and Robust QoS Prediction for Service Adaptation in the Cloud

Jieming Zhu, Pinjia He, Zibin Zheng, and Michael R. Lyu
The Chinese University of Hong Kong

Introduction

Service adaptation is recognized as a key solution for cloud applications to remain resilient against the QoS variations of their component services, whereby QoS requirements can be guaranteed. To make timely and accurate adaptation decisions, effective QoS prediction is desired to obtain current QoS values of component services, but it has not been well studied by the previous work. To bridge this gap, we propose a novel QoS prediction approach, namely adaptive matrix factorization (AMF), which is inspired from the collaborative filtering model in recommender systems. Specifically, our AMF approach extends conventional matrix factorization into an online, accurate, and robust approach by employing techniques of data transformation, online learning and adaptive weights. To evaluate our approach, extensive experiments are conducted based on a real-world large-scale QoS dataset of Web services, and the results show the high performance of our approach in terms of accuracy, robustness, and efficiency. Additionally, a case study on a prototype application demonstrates the benefit of our QoS prediction approach for runtime service adaptation.

Dataset

[Download here][Find more datasets: WS-DREAM]
The main goal of this dataset is to offer real-world data for future research. We have collected 4,500 Web services from the Internet. 142 distributed computers from Planet-Lab have been employed as service users to evaluate the QoS performance of Web services over 64 time slices (15 minute interval).

This data package includes:
1. "rtData" (492 MB): response-time values of 4,500 Web services when invoked by 142 service users over 64 time slices. The data format is as follows:
User ID    Service ID    Time Interval ID    Response Time (s)
e.g.: 98    4352    33    0.311
2. "tpData" (585 MB): throughput values of 4,500 Web services when invoked by 142 service users over 64 time slices. The data format is as follows:
User ID    Service ID    Time Interval ID    Throughput (kbps)
e.g.: 91    1196    62    32.88
3. "readme.txt" (2 KB): This file decribes the dataset in detail.
